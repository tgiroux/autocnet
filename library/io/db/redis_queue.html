io.db.redis_queue — Redis Queue Management¶

The io.db.redis_queue module centralizes methods for accessing the Redis queue.

New in version 0.1.0.

autocnet.io.db.redis_queue.finalize(response, remove_key, queue, outqueue, removequeue)[source]¶

Given a successful processing run, finalize the job in both processing queues

Parameters

  • response (dict) – The reponse to the callback function

  • remove_key (dict) – The key to remove from the removequeue

  • queue (obj) – A Redis queue object

  • outqueue (str) – The name of the redis queue to push the reponse to

  • removequeue (str) – The name of the queue to remove a reponse

autocnet.io.db.redis_queue.pop_computetime_push(queue, inqueue, outqueue)[source]¶

Pop a message from a ‘todo’ queue, compute the maximum possible walltime, push the updated message to ‘processing queue’, and return the original message.’

Parameters

  • queue (object) – A Redis queue object

  • inqueue (str) – The key for the redis store to pop from

  • outqueue (str) – The key for the redis store to push to

Returns

msg – The message from the processing queue.

Return type

dict
